Creating a Web Page - Notes from Session 1

1. The Difference between HTML and CSS

The House Analogy

HTML is the structure and CSS is the Style. Visualize a house - HTML represents the rooms and CSS represents the design elements, like carpeting or the colors of the walls for example.

2. Think Inside the Box

When working on a new design, think in terms of boxes. Everything on a website is really a box.
Even a circle is really a box!

3. The Importance of Structure

As stated above, it's very helpful to think of a webpage as a set of boxes. Oftentimes boxes inside of boxes. This "family tree" structure allows the programmer to breakdown a design and compartamentalize it so it become easier to program.

4. Repetition is bad, bad, bad!

Repetition can lead to errors when there becomes so much repetitive code that the programmer is likely to make mistakes when making changes. Avoiding repetition by defining classes also allows the programmer to easily establish consistency and make changes versus having to manually make lots of updates.

5. Using different Code Editors

1. Scratchpad

Scratchpad is great for beginners. Feedback is immediate. Can't share your work though.

2. Codepen

Codepen has less beginner friendly features but allows all three languages, HTML, CSS, and Java. Codepen is also great for sharing - just need an account.

3. Sublime

Sublime is more difficult to learn but is more robust than Scratchpad or Codepen. It will be the main tool we use as our skills progress.
